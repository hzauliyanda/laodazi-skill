# 老达子风格润色器

对已有历史类文章进行润色，使其符合"老达子"的写作风格。

## 使用方法

```
/laodazi-history-polish <文件路径>
```

润色完成后自动保存，覆盖原文件。

## 文件读写规则（重要）

**必须使用脚本读取和写入 markdown 文件**，不要用 Read/Write 工具直接操作。

**读取文件**：
```bash
bun run ~/.claude/skills/laodazi-history-polish/scripts/polish.ts read <文件路径>
```

**写入文件**（通过 stdin 管道传入内容）：
```bash
cat <<'HEREDOC' | bun run ~/.claude/skills/laodazi-history-polish/scripts/polish.ts write <文件路径>
<润色后的文章内容>
HEREDOC
```

**原因**：Read/Write 工具在处理中文内容和中文标点符号（如""）时可能出现编码偏差，脚本使用 Buffer + TextDecoder/TextEncoder 确保 UTF-8 编码正确处理。

## 润色检查清单

按以下顺序逐项检查并修改：

### 第一步：标点符号（最优先）

**所有标点必须使用中文标点**，尤其是引号。

| 错误 | 正确 |
|------|------|
| "" （英文双引号） | ""（中文双引号） |
| '' （英文单引号） | ''（中文单引号） |
| : （英文冒号） | ：（中文冒号） |
| , （英文逗号） | ，（中文逗号） |
| ; （英文分号） | ；（中文分号） |
| ! （英文感叹号） | ！（中文感叹号） |
| ? （英文问号） | ？（中文问号） |
| () （英文括号） | （）（中文括号） |

**破折号（——）要节制使用**：能用逗号、冒号、句号的地方就不要用破折号。每篇文章最多保留2-3个，只在真正需要强调转折或补充说明的关键位置使用。常见替换：
- 解释说明：用逗号或括号代替（如"他——一个逃犯——竟然称帝了"改为"他，一个逃犯，竟然称帝了"）
- 话题转折：用句号断开或用"但是""不过"连接
- 总结收束：直接用句号

### 第二步：口语化替换

**核心原则：如果这个词你不会在跟朋友聊天时说出来，就不要写进文章里。**

以下替换规则根据上下文灵活运用，不是机械替换：

| 书面语（避免） | 口语化（使用） | 原则 |
|---|---|---|
| 极为 | 非常 | "极为"→"非常" |
| 起初 | 刚开始 | 文言单音→口语双音 |
| 并非如此 | 并不是这样 | 文言简洁→口语完整 |
| 皆言 | 都说 | 文言虚词→大白话 |
| 极大 | 非常大 | "极"→"非常大/特别大" |
| 无人知晓 | 没人知道 | 文言否定→口语否定 |
| 更要命的是 | 更关键的是 | 夸张→平实 |
| 仅仅因为 | 就因为 | 书面强调→口语自然 |
| 数倍 | 好几倍 | 文言量词→口语量词 |
| 其核心 | 它的核心 | "其"→"它" |
| 皆由她出 | 都是由她出的 | 文言省略→口语补全 |
| 仅限于 | 也就是 | 书面限定→口语解释 |
| 并非 | 并不是 | |
| 皆 | 都 | |
| 其 | 它/它的 | |
| 至今未解 | 到现在也没人知道 | |
| 令人唏嘘 | 让人唏嘘不已 | |
| 历史的吊诡之处就在于 | 历史最讽刺的地方就在这 | 文言书面→口语直白 |
| 不胜枚举 | 多得数不过来 | |
| 由此可见 | 所以你看 | |
| 毋庸置疑 | 没得说/不用怀疑 | |
| 众所周知 | 大家都知道 | |
| 不言而喻 | 不用多说 | |
| 纵观全局 | 回头看整件事 | |
| 由此可见一斑 | 从这就能看出来 | |
| 尤为 | 特别 | |
| 亦 | 也 | |
| 乃 | 是 | |
| 之 | 的 | |
| 何以 | 凭什么/为什么 | |

**特别注意**：不要只查上面的表，要通读全文，把任何带有书面感、文言感、学术感的词都替换成口语表达。

### 第三步：文章结构与格式

检查并调整结构和格式：

**格式规范（必须遵守）：**

1. **标题层级**：所有章节标题统一使用 `#` 一级标题（`# 前言`、`# 一、xxx`、`# 老达子说`），不要用 `###` 或 `##`
2. **阅读信息格式**：字数和阅读时长必须用 `>` 引用格式包裹，例如：
   ```markdown
   文｜老达子

   > 本文共2523字，阅读时长大约6分钟
   >
   ```
3. **前言过渡语**：前言末尾必须加入"今天老达子就来跟大家聊聊"这类过渡句，并以"～"结尾自然引出话题。**前言开篇和转折必须自然**，详见下方"前言写法指南"
4. **文头独立行**："文｜老达子"单独一行，不加任何格式标签

**板块要求：**

| 板块 | 要求 |
|------|------|
| 文头 | "文｜老达子"单独一行 + `>`引用格式的字数和阅读时长 |
| 前言 | 用抓手开场，1-2段内抛出核心悬念，末尾加"今天老达子就来跟大家聊聊……～"过渡语。**前言必须自然流畅，严禁生硬的"引用→翻开XX→真相是"套路。详见下方"前言写法指南"** |
| 正文 | 3-5个主题小节，每个小节用 `#` 一级标题，层层递进 |
| 老达子说 | 简短收尾，直击要害。不要模板化地升华、感悟，1-2段说清楚这篇文章最核心的一个判断就行。可以是一句锋利的总结、一个反问、或一个留白的结尾，干净利落，不拖泥带水 |

**前言写法指南（重要）**

润色前言时，最常见的毛病就是机械地走"引用一段话 → 但翻开XX书 → 真相是"的三段式。这种写法用多了，读者一看开头就知道后面要说什么，完全失去吸引力。

**核心原则：前言要像你在饭桌上跟朋友聊起一个八卦，不是在写论文。**

#### 一、开篇方式要多样，不要每次都用诗词引用

根据文章内容选择最合适的开篇方式，以下几种轮换使用：

| 开篇方式 | 适用场景 | 好的例子 |
|----------|----------|----------|
| **反直觉提问** | 大众有固定认知的话题 | "你知道明朝历史上死得最惨的一批人，不是战场上的武将，而是坐在办公室里写公文的文官吗？" |
| **场景切入** | 有戏剧性事件的文章 | "公元1449年的一天，北京城的朝堂上发生了一件前所未有的事：一群文官，当场把锦衣卫的老大活活打死了。" |
| **数字冲击** | 涉及大规模事件 | "一个案子，杀了三万人。不是战场上死的，是一个一个审出来的，杀出来的。这就是明朝的胡惟庸案。" |
| **矛盾对比** | 有反差感的人物或事件 | "同样是锦衣卫指挥使，有人凌迟处死，有人善终并获追封'三公'——明朝三百年，做到这一点的只有一个人。" |
| **诗词引用**（谨慎使用） | 诗词与文章主题有强烈关联时 | 只有当引用的诗句能让读者立刻产生共鸣或好奇时才用，不要为了"文采"硬塞 |

**判断标准**：如果原文的开篇引用跟文章核心内容关系不大，或者引用之后还要费很多话去解释它的含义，那就果断换掉。好的开篇应该让读者3秒内产生"然后呢？"的冲动。

#### 二、转折要自然，严禁机械套路

**禁止使用的转折方式**（这些已经用烂了，读起来像模板）：
- ~~"但当你翻开《明史》……"~~
- ~~"翻开尘封的史书……"~~
- ~~"拨开历史的迷雾……"~~
- ~~"回到《xx史》的字里行间……"~~
- ~~"当我们放下对X的想象……"~~
- ~~"然而，事实真的如此吗？"~~
- ~~"但历史的真相远比这残酷"~~

**正确的做法：不要刻意制造"反转感"，而是让信息自然流动。** 具体方法：

1. **用细节本身制造反差**，而不是用"但是"硬掰：
   - 差："很多人觉得锦衣卫很帅。但翻开《明史》，真相远比这残酷。"
   - 好："飞鱼服、绣春刀，影视剧里的锦衣卫一个比一个帅。可真实的锦衣卫指挥使马顺，是被一群文官在朝堂上活活打死的。"

2. **用提问代替陈述转折**：
   - 差："然而，事实真的如此吗？"
   - 好："三万人里，有几个是真正想造反的？朱元璋不在乎。他要的不是真相，是一个能让他动手的理由。"

3. **直接用画面或数字切入，不要先铺垫再推翻**：
   - 差："人们常说胡惟庸案是正义的审判。但真相是……"
   - 好："洪武十三年，一个宰相倒台了。然后，三万人跟着他一起死。这个数字里，有开国功臣，有地方官员，有跟胡惟庸连面都没见过的小吏。"

#### 三、前言必须制造悬念

前言的最后一个功能，就是让读者忍不住往下看。好的悬念不是故弄玄虚，而是抛出一个读者暂时无法回答的问题。

**制造悬念的技巧**：
- **抛出反常结果**："这些人手里都握着皇帝的刀，可他们几乎没一个得了好死。唯独一个人例外。"
- **制造信息差**："朱元璋晚年干了一件事，这件事让后来所有读史的人脊背发凉。"
- **设置对比**："最狠的那个，杀人如麻；最圆滑的那个，左右逢源。但你猜，最后谁活得最久？"
- **点到为止**：提到一个关键信息但故意不说完，让读者必须往下读才能知道答案

#### 四、前言过渡句要精简

末尾的过渡句直接切入话题，去掉填充话术：
- 直接说"今天老达子就来跟大家聊聊，[具体话题]～"
- 不要加"抛开X不谈""只翻Y档案""回到Z的字里行间"这类废话

如果原文缺少前言或老达子说板块，要补上。

### 第四步：写作技法

检查以下技法是否充分运用：

#### 祛魅反转（最核心）
每个小节至少有一次"大众以为A → 其实是B"的反转。如果原文只是平铺直叙讲历史，要加入反转结构。常见反转模式：
- "很多人以为……但实际上……"
- "网上有很多说法……但这其实是……"
- "教科书里通常会提到……但如果你去翻看……"

#### 史料引用 + 白话翻译
- 引用史料原文后，必须紧跟口语化翻译
- 例：引用"白骨露于野，千里无鸡鸣"后，接"这就是说一百个人里只有一个活了下来"
- 史料用引用格式 > 标注出处

#### 现代类比
把历史概念翻译成当代人秒懂的说法，每篇至少3处：
- 共产国际指挥 → "远程微操"
- 美洲高产作物传入 → "开了个外挂"
- 神的地位 → "编制内办事员"

**禁止使用公司/职场类比**：不要用"CEO""公司""打工人""老板"等现代职场概念做类比。历史文章的读者期待的是历史代入感，公司类比会破坏沉浸感。优先使用日常生活、市井、军事、游戏等领域的类比。

#### 画面感场景
不要用抽象概括，要写具体画面，每篇至少2处：
- 差："机枪威力很大"
- 好："那枪口喷出的火舌，远处一棵合抱粗的大树在几秒钟内被拦腰打断"

#### 短句炸弹
每篇至少2个独立成段的情感金句：
- "那不是数字，那是一条条鲜活的生命。"
- "一切真理，最终都必须讲方言。"

### 第五步：语言风格

| 要求 | 说明 |
|------|------|
| 口语化叙述 | 用"咱们""你看""你想想""你看看"拉近距离 |
| 读者带入 | 多用"你"，少用"人们""大众" |
| 节奏感 | 长短句交替，关键时刻用极短句收束 |
| 适度幽默 | 偶尔用网络语调节气氛，但不要喧宾夺主 |
| 波浪号 | 适当使用"～"增加语气亲和感 |
| 情绪曲线 | 开头轻松/好奇 → 中间逐步加重 → 结尾升华 |

### 第六步：价值观底色

检查文章是否体现了以下价值观（不要刻意添加，而是确保没有违背）：

- **反浪漫化**：不美化历史，揭露盛世背后的残酷
- **反教条**：强调从实际出发，反对脱离实际的照搬
- **人文关怀**：关注被历史碾压的个体

## 常见问题

### 润色后文章变得太"水"了
口语化不等于口水化。保留信息密度，只是换一种更亲切的表达方式。史料引用、数据支撑、逻辑推演这些干货不能丢。

### 不知道某个词该不该替换
判断标准：想象你在跟一个朋友面对面聊天，你会自然地说出这个词吗？如果不会，就换掉。

### 引号到底用哪种
用中文双引号""，不要用英文双引号""，也不要用「」。

### 润色时改了太多内容怎么办
润色的目标是调整风格，不是重写文章。保留原文的核心观点、论据结构和史料引用，只改表达方式。
